## Code of Conduct

This project adheres to a code of conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior to the project maintainers.

## Getting Started

Ledger is a Flutter-based personal finance tracker. To contribute:

1. Ensure you have a [GitHub account](https://github.com/join)
2. Fork the repository on GitHub
3. Create a new branch for your changes (`git checkout -b feature/your-feature-name`)

## Setting Up the Project

### Prerequisites

- **Flutter SDK** (3.x or later) - [Install Flutter](https://flutter.dev/docs/get-started/install)
- **Dart SDK** (comes with Flutter)
- **Git**

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/your-username/Ledger.git
   cd Ledger
   ```

2. Navigate to the Flutter project:
   ```bash
   cd budget
   ```

3. Install dependencies:
   ```bash
   flutter pub get
   ```

4. Generate code (if using drift/build_runner):
   ```bash
   dart run build_runner build --delete-conflicting-outputs
   ```

5. Run the app:
   ```bash
   flutter run
   ```

### Firebase Setup (Optional)

Ledger uses Firebase for authentication, Firestore sync, and hosting. To enable Firebase features:

1. Create a Firebase project
2. Add your platform-specific config files:
   - `budget/android/app/google-services.json`
   - `budget/ios/Runner/GoogleService-Info.plist`
   - `budget/lib/firebase_options.dart` (generated by FlutterFire CLI)

## Development Workflow

1. **Always work in a feature branch** — never commit directly to `main`
2. **Keep changes focused** — one feature/fix per branch
3. **Write tests** for new functionality
4. **Ensure all tests pass** before opening a PR
5. **Update documentation** when necessary

## Running Tests

```bash
# Run all tests
cd budget
flutter test

# Run tests with coverage
flutter test --coverage

# Run a specific test file
flutter test test/unit/currency_test.dart

# Run tests matching a pattern
flutter test --name "currency"

# Check code analysis
flutter analyze
```

### Test Structure

- `test/unit/` — Pure Dart unit tests (no Flutter dependencies)
- `test/widget/` — Flutter widget tests (requires Flutter environment)
- `test/integration/` — Integration tests (not yet set up)

## Commit Message Conventions

We use [Conventional Commits](https://www.conventionalcommits.org/) for commit messages:

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

### Types

| Type       | Description                                           |
|------------|-------------------------------------------------------|
| `feat`     | A new feature                                         |
| `fix`      | A bug fix                                             |
| `docs`     | Documentation only changes                            |
| `style`    | Code style changes (formatting, missing semicolons)   |
| `refactor` | Code changes that neither fix bugs nor add features   |
| `test`     | Adding or updating tests                              |
| `chore`    | Changes to build process, tooling, dependencies       |
| `ci`       | CI/CD configuration changes                           |

### Examples

```
feat(transactions): add recurring transaction support
fix(wallet): correct balance calculation for deleted transactions
docs(readme): update installation instructions
test(currency): add tests for exchange rate conversion
ci: add Flutter build workflow
```

## Pull Request Process

1. **Create an issue** describing the bug or feature (if one doesn't already exist)
2. **Fork and branch** from `main`
3. **Make your changes** following the code style guidelines
4. **Write or update tests** as needed
5. **Run `flutter analyze`** and ensure no new warnings
6. **Run `flutter test`** and ensure all tests pass
7. **Update documentation** if your changes require it
8. **Create a pull request** using the PR template
9. **Address review feedback** promptly
10. **Merge** after approval

### PR Title Format

Follow the same convention as commit messages:
```
feat(wallet): add multi-currency support
fix(ui): resolve date picker crash on Android 14
```

## Code Style

- Follow the [Dart style guide](https://dart.dev/guides/language/effective-dart/style)
- Follow the [Flutter style guide](https://flutter.dev/docs/development/tools/formatting)
- Use `dart format` to format your code:
  ```bash
  dart format lib/
  ```
- Run the analyzer before committing:
  ```bash
  flutter analyze
  ```
- Prefer `const` constructors where possible
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Reporting Issues

When reporting issues, please include:

- A clear, descriptive title
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots or logs (if applicable)
- Environment information:
  - Flutter version (`flutter --version`)
  - Device/emulator type and OS version
  - App version
